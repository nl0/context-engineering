# Publication History

## Credits

This course was written by [Claude](https://claude.ai) (Anthropic), directed and edited by a human author across multiple sessions using aggressive parallelism — research agents, drafting agents, and review agents running concurrently.

The course outline originates from [Latent Patterns](https://latentpatterns.com/courses/context-engineering). Content was developed through deep research into practitioner experiences, academic papers, and production systems. Over 50 sources are cited throughout.

## Timeline

**March 2026** — Initial publication. 8 modules covering tokens, context windows, the messages array, tool calling, sub-agents, message passing, the Ralph Wiggum Loop, and context management strategies.

**Structural review** — Modules 5-7 reordered based on external feedback to follow a micro-to-macro progression: Sub-Agents (isolate heavy calls) → Message Passing (structure communication) → Ralph Wiggum Loop (reset the whole session). Crash-only software framing added to the Ralph Wiggum Loop. Cross-references tightened: Lost-in-the-Middle linked to Context Manager pattern, compaction foreshadowing, read/write parallelism linked to brownfield challenges.

## Design Decisions

- **Prose over bullets** — textbook-style flowing paragraphs, not slide-deck lists
- **Concrete numbers** — every claim backed by specific data (RULER scores, token counts, dollar costs)
- **Practitioner depth** — failure modes, criticisms, and real-world numbers alongside theory
- **Honest treatment** — includes limitations (METR complexity ceiling, code quality concerns, cost)
